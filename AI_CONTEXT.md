## 已完成功能

- [x] 卡片 CRUD 管理：添加卡片、编辑卡片内容、删除单张卡片。
- [x] 定时弹窗复习：
  - 后端 Tokio 异步线程每秒轮询，计时点到达时弹出并置顶复习窗口（reminder 窗口）。
  - 复习时界面动态隐藏答案，用户点击“查看答案”后，可选择“记得”或“不记得”。
- [x] 冷却与随机避重算法：
  - 冷却机制：排除上一张刚复习过的卡片，绝不连续重复弹出。
  - 权重计算：未冷却卡片的权重为 101 - memory_depth。记忆深度（熟练度）越低，被抽中的几率越高。
  - 退化算法：当所有卡片熟练度均达 100%，所有卡片权重强制设为 1，退化为均匀随机抽取。
- [x] 自动挂起与恢复：复习窗口处于打开状态时，主计时器冻结挂起；窗口正常关闭（提交复习）后重置为新一轮倒计时。
- [x] 休息判定（X 关闭）：用户手动点击 X 关闭复习窗口，视为想休息，系统自动关闭“定时复习”开关，并落盘保存。
- [x] 一键导入多张卡片：
  - 支持解析 .txt 或 .md 文本文件。格式规则：第一行问题，第二行答案，第三行空行分隔。
  - 客户端文件读取后进行行扫描与容错过滤，后端批量写入并在同一批次中将 ID 毫秒级递增，彻底避免主键冲突。
- [x] 批量删除模式：支持多选、全选/全不选以及一键批量删除所选卡片。
- [x] 代码全中文注释：Rust 端代码（规范的 /// 文档注释）和 JS 端代码（规范的 JSDoc 注释）已全部补充完毕。

## 已知问题与踩坑记录 (Key Findings)

1. Tauri v2 权限陷阱：
   在 Tauri v2 中，任何前端需要调用的新 Rust 命令必须显式声明在 permissions/app.toml 下的 commands.allow 列表中。例如新加的批量删除 delete_cards 和批量导入 import_cards 命令。若遗漏配置，前端调用时会直接报错 Command not allowed. Command not found。
2. 卡片批量创建的主键碰撞：
   批量导入时若使用 Local::now().timestamp_millis() 直接作为 ID，循环内快速生成的卡片 ID 会完全一致，从而在保存时因主键重复引起异常。需在循环体内对 now 做 now += 1 毫秒级递增处理以防碰撞。
3. 窗口底色与白屏闪烁：
   复习窗口动态构建生成时，由于 WebView 的初始化耗时，容易出现第一帧“纯白”闪烁。我们在 lib.rs 中通过 .set_background_color(Some(tauri::window::Color(9, 13, 22, 255))) 设置其底色为应用暗黑背景色，彻底消除了视觉闪烁。

## 下一次协作切入点 (Next Steps)

如果您是新接手的 AI 助手，在用户发出新需求时，请按如下步骤切入：
1. 获取最新本地数据：可以通过 get_cards 了解卡片的 JSON 实体定义。
2. 状态维护：所有的定时提醒状态逻辑均维护 in lib.rs 中的 AppState，由互斥锁 Mutex 保护。
3. 安全规范：在修改任何代码后，首先应运行 cargo check 和 cargo test 确保无静态编译错误且单元测试全绿。
4. Git 提交限制：除非用户显式回复 git，否则切勿主动执行 git add/commit/push 操作。
